# 音楽制作ガイド — ポケモンの精神を継承する

このリポジトリで作成・修正する **すべてのBGMと効果音** は、ポケットモンスターの
BGM／効果音の精神を継承すること。新しい曲・SEを足すときも、既存曲を直すときも、
必ず以下の原則に沿わせる。（ユーザー恒久指示：2026-07 時点）

## 継承する「ポケモンの精神」

1. **PSG／チップチューンの音色を核にする**
   - リード・ベース・ハモリは矩形波（可変デューティ 12.5% / 25% / 50%）を基本にする。
     Web Audio では `PeriodicWave` でデューティ矩形波を作る。
   - パッド／やわらかい音は triangle（GBの波形メモリchの代わり）。
   - 打楽器はノイズ（バッファ乱数＋ハイパス/バンドパス）で作る。スネアは極短バースト。

2. **チャンネル分離の規律（GBの4ch精神）**
   - 「メロディ／ベース／ハモリ（アルペジオ）／打楽器」の役割を明確に分ける。
   - 音を盛りすぎない。各要素が聴き取れる隙間を残す。

3. **覚えやすい（口ずさめる）メロディ**
   - フック＋応答、モチーフの反復と変化。コードトーン中心＋弱拍で経過音。
   - 装飾よりも「歌える輪郭」を優先。

4. **快適な音域に置く**
   - リードはおおむね C5〜C6 を中心に。**G6 以上の刺さる高音を多用しない。**
   - 低音は濁らないよう整理する。

5. **ループ前提の構成**
   - 短いイントロ → 本体 → シームレスにループ。
   - A/B の対比や、2周目で要素を足すなどの起伏をつける。

6. **音を歪ませない・出しすぎない（クオリティの土台）**
   - マスターにブリックウォール・リミッター（`DynamicsCompressor` ratio≈20, threshold≈-2）を必ず入れる。
   - ディレイのフィードバックや残響は控えめに（feedback ≈ 0.2 以下）。時間経過で
     エコーが溜まって歪む事故を防ぐ。
   - マスター音量は控えめ（`vol/100 * 0.7` 程度）から始める。

7. **効果音もSEの精神で**
   - 攻撃・決定・キャンセル・カーソル・鳴き声などは、短く・ピッチ包絡（スイープ）と
     ノイズで作る。鳴き声は「1つの基本波形をシード違いでピッチ変化」させる方式。

## 実装メモ（このリポジトリの各ページ共通の作り）
- 音源ファイルは使わず、Web Audio API でリアルタイム合成する（1ファイル完結）。
- `voice()`（デチューン2音＋ローパス＋ADSR、`duty`でPSG矩形波、`fx`でディレイ/残響送り）を基本ブロックにする。
- 先読みスケジューラ（イントロを1回 → 本体をループ）で途切れなく再生する。
- 変更後は必ず、JS構文チェックと Web Audio/DOM モックの実行スモークテストを通す。
